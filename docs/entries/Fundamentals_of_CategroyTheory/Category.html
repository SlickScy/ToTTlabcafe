Remark 1.3.

  • A category 𝒞, in which 𝒞1(X,Y ) is a small set (or has the size of a small set) instead of a proper class for any pair of objects X,Y is called a locally small category.
  • A category 𝒞, in which the collection of objects Obj(𝒞) and the collection of Morphisms Mor(𝒞) is a small set (or has the size of a small set) is called a small category.
  • A category 𝒞, which is not necessarily small is called a large category.

2 Foundational issues

Def. 1.1 refers to collections of objects and collections of morphisms. In a mathematical foundation with proposition-valued equality, Def. 1.1 suffices to define a category. This is the case for set theory, class set theory or extensional type theory. However, for other foundations - with weaker notions of equality - such as homotopy type theory, there are multiple definitions of a category and Def. 1.1 is referred to as a wild category.

3 Size issues

Another problem arising from the ambiguity of the terms collection of objects and collection of morphisms is the problem of size of these collections. In particular, similar to naive set theory, when one refers to the ’category of categroies of type T’ CatT, whose collection ob objects is ’all’ categories, whose objects form a collection of type T, then Obj(CatT) itself can not be of type T. I.e. there is no category of all categories; or there are categories which become too large to be considered categories, if we were to refine the term collection in Def. 1.1 any further. This size issue may be tackled in several ways; One worth outlining here is the approach of using Grothendieck universes. A Grothendieck universe U may be thought of as defining a certain universe of mathematical discourse, e.g. U = Set or U = Class. For any two universes V ⊂ U, a category is said to be V -small, if its collection of objects and morphisms is contained in the universe V , U-large if they are contained in the universe U or U-very large if they are not even contained in U. However, even in the latter case, the formalism of Gorthendieck universes ensures that there is some containing universe of U, such that the considered category can be defined properly if suitable universes of discourse are chosen.

4 Examples

The classic example of a category is Set:

  • Obj(Set): (Small) Sets
  • HomSet(X,Y ): Functions from X to Y
  • The Identity morphism and composition of morphisms are evident.

As variations of this example, typical sets with extra structure form categories:

  • Vect𝔽

    • Obj(VectF): Vector Spaces over some field F
    • Vect1(X,Y ): linear functions f : X → Y
    • The Identity morphisms and composition of morphisms arise from their Set equivalents

  • Grp

    • Obj(Grp): Groups
    • Grp1(X,Y ): group homomorphisms f : X → Y

  • Top

    • Obj(Top): Topological Spaces
    • Top1(X,Y ): continuous functions f : X → Y

  • Diff

    • Obj(Diff): Smooth manifolds
    • Diff1(M,N): smooth maps f : M → N

  • Ring

    • Obj(Ring): rings (with unit)
    • Ring1(S,R): ring homomorphisms f : S → R

The list can obviously be extended. To generalize above pattern, every structure of universal algebra together with their structure preserving homomorphisms forms a category using above patterns. This justifies the original motivation for the term category: all of the above categories encapsulate one ”kind of mathematical structure”. They are also called concrete categories. Apart from concrete categories, there are also categories, which roughly model something like ”states” and ”processes” of some systems.

  • Poset: A Poset (partially ordered set) P can be thought of as a category by

    • Obj(P): Elements of P
    • HomP(X,Y ): has a morphism if X < Y , no morphism else.
    • Composition of morphisms is obvious (as there is only one choice) and corresponds to transitivity of the < relation.
    • The identity morphism for elements X corresponds to transitivity of <.

  • Group: A Group G can be thought of as a category with one object, where its endomorphisms are the group objects and composition of morphisms is given by the group action.

    • Obj(G) = {∗}
    • HomG(∗,∗) = G (as a set)
    • For f,g ∈ HomG(∗,∗), g ∘ f = g ⋅Gf, where ⋅G denotes the group action in G.
    • id∗ = eG, where eG denotes the unit element of the group G.
  • Monoid: Similar to Group: A monoid can be thought of as a category with a single object. Whereas by composition in Group, all morphisms are reversible, this is no longer necessarily the case in Monoid. The definition works the same though. Conversely, categories may be thought of as a ”many-object version” of monoids.
